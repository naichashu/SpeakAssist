## 核心流程

### 流程A：应用内发送指令

```
用户在首页输入文字或点击语音按钮
    ↓
语音识别完成 → 识别文本显示在输入框
    ↓ 用户点击发送
聊天列表显示用户消息
    ↓
执行卡片悬浮窗出现（屏幕中上） + 圆形悬浮窗隐藏
    ↓ AI逐步执行操作，卡片实时更新"当前步骤"
操作完成 or 用户点击取消
    ↓
执行卡片显示最终结果 → 保留5秒后消失
    ↓
聊天列表显示系统消息（成功/失败）
    ↓
圆形悬浮窗重新出现，回到空闲状态
```

### 流程B：悬浮窗快捷指令（自动发送，不需要确认）

```
空闲状态（圆形悬浮窗显示在屏幕边缘）
    ↓ 用户点击悬浮窗
悬浮窗展开为长条形，开始语音识别
    ↓ 识别中，实时显示文字
识别成功：展开区域显示识别文本 → 自动发送执行
    ↓
悬浮窗收回为圆形 → 执行卡片悬浮窗出现
    ↓ AI逐步执行操作
（后续同流程A）

识别失败：展开区域显示"识别失败" → 3秒后自动收回为圆形
```

```

├─────────────────────────┤
│ ⚠️ 权限提示（可收起）    │  <- 权限信息条（权限缺失时显示）
├─────────────────────────┤
│                         │
│   聊天消息列表          │  <- 主要内容区：消息滚动列表
│   - 用户消息            │     用户指令（右靠，蓝色背景）
│   - 系统反馈            │     系统消息（左靠，灰色背景）
│                         │
│                         │
├─────────────────────────┤
│ [聊天框...........]  🎙️  │  <- 底部输入区：输入框 + 语音按钮
│                  [发送]  │                      + 发送按钮

浮层元素（系统悬浮窗，覆盖在其他App之上）：
```

#### 空状态
- 聊天列表为空时，显示引导提示，如："点击语音按钮或输入文字，开始你的第一个指令"

#### 权限提示条
- **位置**: 聊天列表上方，顶部导航下
- **显示条件**: 任一必要权限缺失时显示
- **内容**:
  ```
  ⚠️ 缺少权限：无障碍服务、录音权限
  [前往设置] [暂时忽略]
  ```
- **操作**: 点击"前往设置"跳转到本App的设置页面；点击"暂时忽略"关闭提示

#### 聊天消息区
**用户消息**:
```
                ┌─────────────────┐
                │  打开微信        │  <- 右靠，蓝色背景，圆角
                └─────────────────┘
```

**系统消息**:
```
┌──────────────────────┐
│ 成功打开微信，已进入 │  <- 左靠，灰色背景，圆角
│ 聊天列表页面         │
└──────────────────────┘
```

#### 底部输入区
- **左**: 聊天输入框（可手动输入文本）
- **右侧1**: 语音按钮（点击启动语音识别，识别结果填入输入框，用户手动发送）
- **右侧2**: 发送按钮（点击发送消息）
- 布局: 水平排列

### 2. 圆形悬浮窗（系统悬浮窗）

#### 空闲状态（圆形）
```
    [🔵]   <- 圆形，半隐藏在屏幕右侧边缘
```
- **形状**: 圆形
- **大小**: ~60dp 直径
- **位置**: 屏幕右侧边缘，可半隐藏在屏幕外
- **显示条件**: 空闲状态，且用户在设置中开启了悬浮窗功能
- **视觉样式**: 渐变色圆形，带阴影
- **消失条件**: 任务执行开始时隐藏

#### 展开状态（语音识别中）
```
    [🔵] 正在听......        <- 点击后展开为长条形
    [🔵] 打开微信发消息      <- 识别完成，显示识别文本
    [🔵] 识别失败            <- 识别失败时显示
```
- **触发**: 点击圆形悬浮窗
- **展开方向**: 向左展开为圆角长条形
- **最左侧**: 保留原悬浮窗Logo图标
- **右侧**: 显示识别状态/识别文本
- **交互**:
  - 识别中：右侧显示"正在听......"（可带波形动画）
  - 识别成功：右侧显示识别文本 → **自动发送执行**（无需用户确认） → 悬浮窗收回圆形
  - 识别失败：右侧显示"识别失败" → **3秒后自动收回**圆形
  - 点击左侧Logo：**取消识别**，立即收回圆形

### 3. 执行卡片（系统悬浮窗）

- **实现方式**: 系统悬浮窗（WindowManager），覆盖在其他App之上
- **位置**: 屏幕中上方（尽量往上），初始固定
- **大小**: 约宽度 80-90% 屏幕宽度，高度由内容撑开
- **背景**: 半透明（可以看到身后的App）
- **可拖动**: 是，用户可以拖动改变位置
- **显示条件**: 任务执行过程中显示
- **内容**:
  ```
  ┌────────────────────────────┐
  │ 正在执行：打开微信         │  <- 标题：任务目标
  ├────────────────────────────┤
  │ 已执行第 3 步              │  <- 进度：仅显示当前步数
  ├────────────────────────────┤
  │ 当前步骤：点击应用图标     │  <- 实时操作：现在在做什么
  │ 等待响应...               │  <- 状态：是否正在等待
  ├────────────────────────────┤
  │          [取消任务]         │  <- 控制按钮
  └────────────────────────────┘
  ```
- **步数显示**: 仅显示已执行步数（如"第3步"），不显示总步数（总步数无法预知）
- **内容说明**: 显示"现在在干什么"，不显示AI的思考过程
- **更新频率**: 实时更新"当前步骤"和"等待状态"
- **消失条件**: 任务完成或用户取消后，保留5秒，然后消失

## 交互细节

### 语音输入流程（应用内）
```
用户点击底部输入区的语音按钮
    ↓
显示"正在听......"（或波形动画）
    ↓
用户说话
    ↓
调用百度语音识别API
    ↓
识别成功 → 识别文本填入输入框
    ↓
用户手动点击发送按钮
    ↓
消息进入聊天列表，开始执行
```

### 语音输入流程（悬浮窗快捷）
```
用户点击圆形悬浮窗
    ↓
悬浮窗展开为长条形，左侧Logo + 右侧"正在听......"
    ↓
用户说话
    ↓
调用百度语音识别API
    ↓
识别成功 → 展开区域显示识别文本 → 自动发送执行（无需确认）
    ↓
悬浮窗收回圆形 → 执行卡片出现
```

**取消操作**: 识别过程中，点击左侧Logo可取消，悬浮窗立即收回圆形

**识别失败**: 展开区域显示"识别失败"，3秒后自动收回圆形

### 权限缺失提示
- 打开App时，检查必要权限（无障碍服务、输入法服务、录音权限、悬浮窗权限）
- 如果缺失，在首页顶部显示权限提示条
- 用户点击"前往设置" → 跳转到本App的设置页面
- 用户点击"暂时忽略" → 关闭提示

### 执行卡片操作
- **拖动**: 按住卡片任意位置可拖动，改变位置
- **取消**: 点击"取消任务"按钮，停止当前执行，卡片5秒后消失
- **自动消失**: 任务完成（成功或失败）5秒后，卡片自动消失

### 截屏处理
- 执行过程中需要截屏时：
  1. 隐藏执行卡片悬浮窗
  2. 进行截屏
  3. 截屏完成后，恢复执行卡片悬浮窗
- 圆形悬浮窗在执行期间本身就是隐藏的，无需额外处理

### 网络异常处理
- API调用超时或网络断开时，执行卡片显示错误信息
- 聊天列表显示失败消息，附带简要错误原因
- 语音识别网络异常按识别失败处理

### 消息显示规则
- **用户消息**: 输入框发送的内容，显示在右侧，蓝色背景
- **系统消息**: AI执行后的最终结果
  - 成功消息：显示执行完成和简要结果
  - 失败消息：显示失败原因
- **不显示中间过程**: 聊天列表只显示用户指令和最终结果，中间步骤在历史详情页查看

## 数据存储

### Room 数据库
用于存储聊天历史记录，支持侧栏历史列表和历史详情页。

**主要表结构**:

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| `task_sessions` | 任务会话 | id, user_command, status(成功/失败/执行中), created_at |
| `task_steps` | 任务步骤 | id, session_id(外键), step_number, action_type, action_description, ai_thinking, created_at |

- 每次用户发送指令创建一条 `task_sessions` 记录
- 每执行一步创建一条 `task_steps` 记录
- 侧栏查询 `task_sessions` 列表
- 详情页查询某 session 下的所有 `task_steps`

### DataStore Preferences
用于存储简单的App设置项（依赖已存在于项目中）。

| 键 | 类型 | 用途 |
|----|------|------|
| `floating_window_enabled` | Boolean | 悬浮窗功能开关 |
