## 1. 问题定义

### 1.1 当前实现

项目有两处音频处理场景，都用固定 RMS 阈值：

**场景 A：唤醒词监听**（`WakeWordListeningManager`）
- Mic → AudioRecord (16kHz PCM) → Vosk Recognizer (grammar 约束到唤醒词白名单) → onWakeWordDetected
- RMS 阈值固定 500，仅用于重置超时计时器
- Vosk grammar 本身就是"人声过滤器"，约束输出到白名单或 `[unk]`

**场景 B：语音输入**（`BaiduSpeechManager`）
- Mic → AudioRecord (16kHz PCM) → 实时 RMS + 静音 VAD → 全量 PCM Base64 → Baidu ASR API
- RMS 阈值固定 800，持续静音 1.5s 自动停止

### 1.2 真实问题（基于代码审查）

| 编号 | 问题 | 场景 | 严重程度 | 根因 |
|------|------|------|----------|------|
| P0-1 | 安静环境下唤醒词漏检 | 唤醒 | 高 | 固定 RMS 阈值在安静环境下误把底噪当成"有声音"，打乱 Vosk 识别节奏 |
| P0-2 | 嘈杂环境下云端识别准确率下降，用户无感知 | 语音输入 | 高 | 固定 VAD 参数在嘈杂环境阈值过低，噪声被送到百度；用户不知道环境不适合说话 |
| P1-1 | 用户从安静环境走到嘈杂环境后阈值不跟随 | 两者 | 中 | 阈值固定无法跟随环境变化 |
| P1-2 | 机械噪声 / 风声等特殊噪声未被识别 | 两者 | 中 | RMS 无法区分噪声类型，无法给用户正确反馈 |

### 1.3 关键认知纠正

**初版方案（v1）认为应该在 Vosk 前做 Layer 1 过滤跳过噪声帧——这是错误的**。

Vosk 的 `acceptWaveForm` 需要每一帧音频才能维持识别器的状态机；跳帧会破坏识别节奏甚至导致漏检。**grammar 约束本身就是人声过滤器**，噪声进不来才是正常行为。

正确定位：

```
唤醒词监听：核心矛盾 = 安静环境漏检 + 阈值不适应（不是误检也不是 CPU）
语音输入：   核心矛盾 = 嘈杂环境识别质量下降 + 用户无感知
共同矛盾：   固定阈值不适应环境变化
```

## 2. 设计原则

1. **不跳 Vosk 帧**：唤醒场景保留全帧 Vosk 处理，只做能量判断不做过滤
2. **环境预学习**：每次开录前 500ms 采集环境噪声，测量基底，据此初始化 VAD
3. **自适应基底**：运行中通过指数移动平均（EMA）持续跟踪噪声基底变化
4. **用户感知**：语音输入时 UI 实时反馈噪声级别
5. **零新依赖**：全部用纯 Kotlin 算法（STE + ZCR + 低频能量比），不引入 TFLite/JNI/Native 库

## 3. 整体架构

### 3.1 分层策略

```
AudioSource.MIC (16kHz PCM)
            │
            ▼

  │ ┌──────────────┐ │
  │ │Layer 1: 能量 │ │ STE + AdaptiveVad 基底跟踪
  │ │判断是否语音  │ │ 用途：UI / 计时器 / VAD 停止判定
  │ └──────────────┘ │
```

## 4. 核心模块详细设计

### 4.1 AdaptiveVad — 自适应语音活动检测器

**核心算法**：
- 用短时能量 STE（平方和，不开方）代替 RMS，省一次 sqrt
- 用指数移动平均（EMA）跟踪噪声基底
- 只在连续确认无语音时更新基底，防止人声污染

```kotlin
package com.example.speech

import android.util.Log
import kotlin.math.sqrt

/**
 * 自适应语音活动检测器。
 *
 * 通过 EMA 跟踪环境噪声基底，动态调整语音判定阈值。
 * 噪声基底只在连续确认无语音时更新（避免人声污染基底）。
 *
 * @param alpha                EMA 系数，越小基底更新越慢（默认 0.05）
 * @param snrThreshold         信噪比系数（默认 3.5，~11dB）
 * @param confirmSilenceFrames 连续多少帧无语音才更新基底（默认 15 ~500ms）
 */
class AdaptiveVad(
    private val alpha: Double = 0.05,
    private val snrThreshold: Double = 3.5,
    private val confirmSilenceFrames: Int = 15
) {

    companion object {
        private const val TAG = "AdaptiveVad"

        /** 计算短时能量（不开方，等价于 RMS²）。 */
        fun calculateSTE(buffer: ByteArray, read: Int): Double {
            var sum = 0.0
            var i = 0
            while (i + 1 < read) {
                val sample = ((buffer[i].toInt() and 0xFF) or
                    (buffer[i + 1].toInt() shl 8)).toShort().toInt()
                sum += sample.toDouble() * sample.toDouble()
                i += 2
            }
            return sum
        }

        /** STE → RMS。 */
        fun steToRms(ste: Double, sampleCount: Int): Double =
            if (sampleCount > 0) sqrt(ste / sampleCount) else 0.0
    }

    private var noiseFloor = 100.0
    private var silenceFrameCount = 0

    fun isSpeechCandidate(ste: Double): Boolean = ste > noiseFloor * snrThreshold

    fun updateNoiseFloor(ste: Double): Boolean {
        if (ste < noiseFloor * snrThreshold) {
            silenceFrameCount++
            if (silenceFrameCount >= confirmSilenceFrames) {
                noiseFloor = (1 - alpha) * noiseFloor + alpha * ste
                silenceFrameCount = 0
                Log.v(TAG, "噪声基底更新: ${noiseFloor.toInt()} STE")
                return true
            }
        } else {
            silenceFrameCount = 0
        }
        return false
    }

    fun setInitialNoiseFloor(steFloor: Double) { noiseFloor = steFloor }
    fun reset(baseLevel: Double = 100.0) {
        noiseFloor = baseLevel
        silenceFrameCount = 0
    }

    fun getNoiseFloorSte(): Double = noiseFloor
    fun getNoiseFloorRms(sampleCount: Int = 512): Double =
        steToRms(noiseFloor, sampleCount)
    fun getSpeechThresholdRms(sampleCount: Int = 512): Double =
        steToRms(noiseFloor * snrThreshold, sampleCount)
}
```

### 4.2 SpectralNoiseDetector — 频域噪声检测器

用轻量时域特征近似频域分析，**不做 FFT**（移动端每帧 FFT 成本太高）。

```kotlin
package com.example.speech

import kotlin.math.abs

/**
 * 频域噪声检测器。
 * 通过 ZCR + 低频能量比识别噪声类型，用于 UI 提示。
 *
 * 算法原理：
 * - 过零率（ZCR）：相邻样本符号变化的比率
 *   人声 3-10%，机械噪声 <2%，风声 >30%
 * - 低频能量比：用样本差分近似高频分量
 *   低频能量 = 总能量 - 高频能量
 *
 * @param historySize 滑动窗口大小（帧数），控制平滑程度
 */
class SpectralNoiseDetector(private val historySize: Int = 20) {

    private val zcrHistory = ArrayDeque<Double>()
    private val lfRatioHistory = ArrayDeque<Double>()

    fun calculateZcr(buffer: ByteArray, read: Int): Double {
        var zcr = 0
        var prevSample = 0
        var i = 0
        while (i + 1 < read) {
            val sample = ((buffer[i].toInt() and 0xFF) or
                (buffer[i + 1].toInt() shl 8)).toShort().toInt()
            if (prevSample xor sample < 0) zcr++
            prevSample = sample
            i += 2
        }
        val sampleCount = read / 2
        return if (sampleCount > 0) zcr.toDouble() / sampleCount else 0.0
    }

    fun calculateLowFrequencyRatio(buffer: ByteArray, read: Int): Double {
        var highFreqEnergy = 0L
        var totalEnergy = 0L
        var prevSample = 0
        var i = 0
        while (i + 1 < read) {
            val sample = ((buffer[i].toInt() and 0xFF) or
                (buffer[i + 1].toInt() shl 8)).toShort().toInt()
            totalEnergy += sample.toLong() * sample.toLong()
            if (i > 0) {
                val diff = (sample - prevSample).toLong()
                highFreqEnergy += diff * diff
            }
            prevSample = sample
            i += 2
        }
        return if (totalEnergy > 0)
            1.0 - highFreqEnergy.toDouble() / totalEnergy
        else 0.5
    }

    fun analyze(buffer: ByteArray, read: Int): NoiseReport {
        val zcr = calculateZcr(buffer, read)
        val lfRatio = calculateLowFrequencyRatio(buffer, read)
        zcrHistory.addLast(zcr)
        lfRatioHistory.addLast(lfRatio)
        if (zcrHistory.size > historySize) zcrHistory.removeFirst()
        if (lfRatioHistory.size > historySize) lfRatioHistory.removeFirst()

        val avgZcr = zcrHistory.average()
        val avgLfRatio = lfRatioHistory.average()
        return NoiseReport(
            zcr = avgZcr,
            lowFreqRatio = avgLfRatio,
            noiseType = classifyNoise(avgZcr, avgLfRatio),
            sampleCount = zcrHistory.size
        )
    }

    private fun classifyNoise(avgZcr: Double, avgLfRatio: Double): NoiseType = when {
        avgZcr < 0.02 && avgLfRatio > 0.75 -> NoiseType.MECHANICAL
        avgZcr > 0.30 -> NoiseType.WIND_WHITE
        avgZcr in 0.03..0.10 && avgLfRatio > 0.60 -> NoiseType.CROWD
        else -> NoiseType.QUIET
    }

    fun reset() {
        zcrHistory.clear()
        lfRatioHistory.clear()
    }
}

data class NoiseReport(
    val zcr: Double,
    val lowFreqRatio: Double,
    val noiseType: NoiseType,
    val sampleCount: Int
) {
    /** 历史窗口未满时不可靠。 */
    fun isReliable(): Boolean = sampleCount >= 10
}

enum class NoiseType {
    QUIET,        // 安静或轻度噪声
    MECHANICAL,   // 机械噪声（空调/风扇/引擎）
    WIND_WHITE,   // 风声或白噪声
    CROWD         // 背景人声嘈杂
}
```

### 4.3 EnvironmentAnalyzer — 环境分析器（封装层）

统一接口，封装预学习逻辑和两层检测。

```kotlin
package com.example.speech

import android.util.Log
import kotlinx.coroutines.flow.MutableStateFlow
import kotlinx.coroutines.flow.StateFlow
import kotlinx.coroutines.flow.asStateFlow

/**
 * 环境分析器。封装 AdaptiveVad + SpectralNoiseDetector，
 * 提供统一的预学习与实时分析接口。
 *
 * @param preLearningFrames 预学习帧数（每帧 ~32ms @ 16kHz/512samples，16 帧 ≈ 512ms）
 */
class EnvironmentAnalyzer(
    private val preLearningFrames: Int = 16
) {

    companion object {
        private const val TAG = "EnvironmentAnalyzer"
    }

    private val adaptiveVad = AdaptiveVad()
    private val spectralDetector = SpectralNoiseDetector()

    private var preLearningFrameCount = 0
    private var preLearningSteSum = 0.0
    private var isPreLearning = false
    private var initialNoiseFloorRms = 0.0

    private val _noiseLevel = MutableStateFlow(NoiseLevel.LOW)
    val noiseLevel: StateFlow<NoiseLevel> = _noiseLevel.asStateFlow()

    /** 开始环境预学习。调用后预学习帧用于测基底，不做语音判断。 */
    fun startPreLearning() {
        preLearningFrameCount = 0
        preLearningSteSum = 0.0
        isPreLearning = true
        adaptiveVad.reset(100.0)
        spectralDetector.reset()
        _noiseLevel.value = NoiseLevel.LOW
        Log.d(TAG, "开始环境预学习")
    }

    /**
     * 处理一帧音频。
     * @return 预学习阶段返回 null；正常阶段返回 SpeechFrameResult
     */
    fun processFrame(buffer: ByteArray, read: Int): SpeechFrameResult? {
        return if (isPreLearning) {
            processPreLearningFrame(buffer, read)
            null
        } else {
            processNormalFrame(buffer, read)
        }
    }

    private fun processPreLearningFrame(buffer: ByteArray, read: Int) {
        val ste = AdaptiveVad.calculateSTE(buffer, read)
        preLearningSteSum += ste
        preLearningFrameCount++
        if (preLearningFrameCount >= preLearningFrames) {
            val avgSte = preLearningSteSum / preLearningFrames
            adaptiveVad.setInitialNoiseFloor(avgSte)
            initialNoiseFloorRms = adaptiveVad.getNoiseFloorRms(read / 2)
            isPreLearning = false
            Log.d(TAG, "环境预学习完成，噪声基底 RMS = " +
                "${"%.1f".format(initialNoiseFloorRms)}")
        }
    }

    private fun processNormalFrame(buffer: ByteArray, read: Int): SpeechFrameResult {
        val ste = AdaptiveVad.calculateSTE(buffer, read)
        val isSpeech = adaptiveVad.isSpeechCandidate(ste)
        adaptiveVad.updateNoiseFloor(ste)
        val noiseReport = spectralDetector.analyze(buffer, read)
        updateNoiseLevel(ste, read, noiseReport)
        return SpeechFrameResult(ste, isSpeech, noiseReport)
    }

    private fun updateNoiseLevel(ste: Double, read: Int, report: NoiseReport) {
        val sampleCount = read / 2
        val rms = AdaptiveVad.steToRms(ste, sampleCount)
        val floorRms = adaptiveVad.getNoiseFloorRms(sampleCount)
        val snr = if (floorRms > 0) rms / floorRms else 1.0

        _noiseLevel.value = when {
            !report.isReliable() -> NoiseLevel.LOW   // 数据不足，不警告
            report.noiseType == NoiseType.MECHANICAL ||
                report.noiseType == NoiseType.WIND_WHITE -> NoiseLevel.HIGH
            snr > 10 -> NoiseLevel.LOW
            snr > 3 -> NoiseLevel.MEDIUM
            else -> NoiseLevel.HIGH
        }
    }

    fun getAdaptiveVad(): AdaptiveVad = adaptiveVad
    fun getInitialNoiseFloorRms(): Double = initialNoiseFloorRms
    fun isPreLearningInProgress(): Boolean = isPreLearning

    /** 场景切换时重置（重新预学习）。 */
    fun reset() = startPreLearning()
}

data class SpeechFrameResult(
    val ste: Double,
    val isSpeechCandidate: Boolean,
    val noiseReport: NoiseReport
)

/** UI 用噪声级别。 */
enum class NoiseLevel {
    LOW,     // 安静，绿色
    MEDIUM,  // 一般，黄色
    HIGH     // 嘈杂，红色 + 提示
}
```

## 5. 场景 A：唤醒词监听改造

### 5.1 目标
解决安静环境下唤醒词漏检，同时避免嘈杂环境误触发。

### 5.2 改造要点
1. `startWakeWordLoop()` 前插入 ≤ 2s 环境预学习（500ms 完成）
2. 主循环用 `AdaptiveVad.isSpeechCandidate()` 替代 `rms > 500` 重置超时计时
3. **Vosk 始终接收完整帧，不跳过**
4. 唤醒词匹配置信度根据噪声基底动态调整（嘈杂放宽、安静收紧）

### 5.3 改造后的流程

```
startListening()
  └─ 启动 AudioRecord
  └─ environmentAnalyzer.startPreLearning()
  └─ 预学习循环（最多 2s，正常 ~500ms 完成）
       └─ environmentAnalyzer.processFrame() 累积基底
  └─ 主循环
       └─ environmentAnalyzer.processFrame() → SpeechFrameResult
       └─ if (isSpeechCandidate) 重置超时计时器
       └─ recognizer.acceptWaveForm(buffer, read)  ← 永不跳过
       └─ partialResult → containsWakeWord(text, noiseFloorRms)
            └─ 是 → onWakeWordDetected
            └─ 否 → 继续
```

### 5.4 关键代码

```kotlin
// WakeWordListeningManager.kt 新增字段
private val environmentAnalyzer = EnvironmentAnalyzer()

private suspend fun startWakeWordLoop() {
    // ... 原有 Recognizer/AudioRecord 初始化 ...
    val buffer = ByteArray(bufferSize)

    // === 环境预学习阶段 ===
    environmentAnalyzer.startPreLearning()
    val preLearningStart = System.currentTimeMillis()
    while (audioRecord?.recordingState == AudioRecord.RECORDSTATE_RECORDING &&
        environmentAnalyzer.isPreLearningInProgress() &&
        System.currentTimeMillis() - preLearningStart < 2000) {
        val read = audioRecord?.read(buffer, 0, buffer.size) ?: -1
        if (read <= 0) { delay(10); continue }
        environmentAnalyzer.processFrame(buffer, read)
        delay(10)
    }
    val noiseFloorRms = environmentAnalyzer.getInitialNoiseFloorRms()
    Log.d(TAG, "唤醒预学习完成，基底 RMS=${"%.1f".format(noiseFloorRms)}")

    // === 主循环 ===
    var listeningStartTime = System.currentTimeMillis()
    var hasSpeechDetected = false

    loop@ while (audioRecord?.recordingState == AudioRecord.RECORDSTATE_RECORDING) {
        val read = audioRecord?.read(buffer, 0, buffer.size) ?: -1
        if (read <= 0) { delay(10); continue }

        // Layer 1/2：判断但不过滤
        val frameResult = environmentAnalyzer.processFrame(buffer, read)
        if (frameResult?.isSpeechCandidate == true) {
            hasSpeechDetected = true
            listeningStartTime = System.currentTimeMillis()
        }

        // Vosk 永远接收完整帧
        recognizer.acceptWaveForm(buffer, read)
        val partialResult = voskManager?.getPartialResult().orEmpty()

        if (partialResult.isNotBlank() &&
            containsWakeWord(partialResult, noiseFloorRms)) {
            Log.i(TAG, "唤醒词检测到: $partialResult")
            recognizer.reset()
            handler.post { listener?.onWakeWordDetected() }
            updateState(State.WAKE_DETECTED)
            break@loop
        }

        if (hasSpeechDetected &&
            System.currentTimeMillis() - listeningStartTime > MAX_LISTENING_MS) {
            recognizer.reset()
            hasSpeechDetected = false
            listeningStartTime = System.currentTimeMillis()
        }
        delay(10)
    }
    stopRecording()
}

/** 匹配置信度根据环境噪声动态调整。 */
private fun containsWakeWord(text: String, noiseFloorRms: Double): Boolean {
    val confidenceThreshold = when {
        noiseFloorRms > 300 -> 0.7   // 嘈杂：70% 字符匹配即触发
        noiseFloorRms > 150 -> 0.8
        else -> 0.9                  // 安静：90% 匹配，减少误检
    }
    return matchWithConfidence(text, WAKE_WORDS, confidenceThreshold)
}

private fun matchWithConfidence(
    text: String,
    words: List<String>,
    threshold: Double
): Boolean {
    val cleanText = text.replace(Regex("[\\s\\p{Punct}]"), "").lowercase()
    for (word in words) {
        val cleanWord = word.replace(Regex("[\\s\\p{Punct}]"), "").lowercase()
        if (cleanText.contains(cleanWord)) return true
        val lcs = longestCommonSubstring(cleanText, cleanWord)
        if (cleanWord.isNotEmpty() &&
            lcs.length.toDouble() / cleanWord.length >= threshold) return true
    }
    return false
}

private fun longestCommonSubstring(a: String, b: String): String {
    if (a.isEmpty() || b.isEmpty()) return ""
    val dp = Array(a.length + 1) { IntArray(b.length + 1) }
    var maxLen = 0
    var endIndex = 0
    for (i in 1..a.length) for (j in 1..b.length) {
        if (a[i - 1] == b[j - 1]) {
            dp[i][j] = dp[i - 1][j - 1] + 1
            if (dp[i][j] > maxLen) { maxLen = dp[i][j]; endIndex = i }
        }
    }
    return a.substring(endIndex - maxLen, endIndex)
}
```

### 5.5 与原有逻辑差异

| 对比项 | 改造前 | 改造后 |
|--------|--------|--------|
| 环境适应 | 无，固定 `rms > 500` | 自适应，跟踪噪声基底 |
| "有声音"判定 | `rms > 500` | `adaptiveVad.isSpeechCandidate()` |
| 唤醒词匹配 | 固定 contains | 噪声基底→置信度→LCS 模糊匹配 |
| 预学习 | 无 | 每次监听前 ≤ 500ms |

## 6. 场景 B：语音输入改造

### 6.1 目标
嘈杂环境下自适应 VAD 参数 + 实时噪声级别反馈到 UI。

### 6.2 动态 VAD 参数映射

| 环境 | 噪声基底 RMS | 能量阈值 | 静音持续 | 最大时长 |
|------|------------|----------|----------|----------|
| 安静 | < 100 | 300 | 1.2s | 30s |
| 一般 | 100-300 | 800 | 1.5s | 45s |
| 嘈杂 | > 300 | 1200 | 2.5s | 60s |

### 6.3 关键代码

```kotlin
// BaiduSpeechManager.kt 新增字段
private val environmentAnalyzer = EnvironmentAnalyzer()

/** 暴露给 FloatingWindowManager 订阅。 */
val noiseLevel: StateFlow<NoiseLevel> get() = environmentAnalyzer.noiseLevel

private suspend fun startRecordingAndRecognize(sessionId: Int) {
    // ... 权限检查、token、AudioRecord 初始化 ...
    audioRecord?.startRecording()
    isListening = true
    isRecording = true
    withContext(Dispatchers.Main) { callback?.onReady() }

    val audioData = ByteArrayOutputStream()
    val buffer = ByteArray(bufferSize)

    // === 环境预学习（500ms）===
    environmentAnalyzer.startPreLearning()
    val preLearningStart = System.currentTimeMillis()
    while (isSessionActive(sessionId) && isRecording &&
        environmentAnalyzer.isPreLearningInProgress() &&
        System.currentTimeMillis() - preLearningStart < 800) {
        val read = audioRecord?.read(buffer, 0, buffer.size) ?: -1
        if (read > 0) environmentAnalyzer.processFrame(buffer, read)
        delay(10)
    }
    val noiseFloorRms = environmentAnalyzer.getInitialNoiseFloorRms()
    val vadParams = selectVadParams(noiseFloorRms)
    Log.d(TAG, "语音输入预学习完成，基底 RMS=${"%.1f".format(noiseFloorRms)}, " +
        "VAD=${vadParams}")

    // === 主录音循环 ===
    val startTime = System.currentTimeMillis()
    var hasSpeech = false
    var silenceStartTime = 0L

    while (isSessionActive(sessionId) && isRecording &&
        System.currentTimeMillis() - startTime < vadParams.maxRecordingDurationMs) {
        val read = audioRecord?.read(buffer, 0, buffer.size) ?: -1
        if (read <= 0) { if (read < 0) break; continue }
        audioData.write(buffer, 0, read)

        val frameResult = environmentAnalyzer.processFrame(buffer, read)
        val ste = frameResult?.ste ?: 0.0
        val rms = AdaptiveVad.steToRms(ste, read / 2)
        val volume = (rms / 32768 * 100).toInt().coerceIn(0, 100)
        withContext(Dispatchers.Main) { callback?.onVolumeChanged(volume) }

        val now = System.currentTimeMillis()
        if (rms > vadParams.energyThreshold) {
            hasSpeech = true
            silenceStartTime = 0L
        } else if (hasSpeech) {
            if (silenceStartTime == 0L) silenceStartTime = now
            val speechDuration = now - startTime
            val elapsedSilence = now - silenceStartTime
            if (speechDuration > vadParams.minSpeechDurationMs &&
                elapsedSilence >= vadParams.silenceDurationMs) {
                Log.d(TAG, "检测到静音 ${elapsedSilence}ms，停止录音")
                isRecording = false
                break
            }
        }
    }
    // ... 后续停止录音、发送百度识别 ...
}

private fun selectVadParams(noiseFloorRms: Double): VadParams = when {
    noiseFloorRms < 100 -> VadParams.QUIET
    noiseFloorRms < 300 -> VadParams.NORMAL
    else -> VadParams.NOISY
}

data class VadParams(
    val energyThreshold: Double,
    val minSpeechDurationMs: Long,
    val silenceDurationMs: Long,
    val maxRecordingDurationMs: Long
) {
    companion object {
        val QUIET = VadParams(300.0, 200L, 1200L, 30_000L)
        val NORMAL = VadParams(800.0, 300L, 1500L, 45_000L)
        val NOISY = VadParams(1200.0, 500L, 2500L, 60_000L)
    }
}
```

### 6.4 与原有逻辑差异

| 对比项 | 改造前 | 改造后 |
|--------|--------|--------|
| 能量阈值 | 固定 800 | 动态 300 / 800 / 1200 |
| 静音判定 | 固定 1.5s | 动态 1.2s / 1.5s / 2.5s |
| 最大录音时长 | 固定 60s | 动态 30s / 45s / 60s |
| 噪声反馈 | 无 | `noiseLevel` StateFlow + `onVolumeChanged` |

## 7. UI 噪声级别反馈

### 7.1 订阅与生命周期

`FloatingWindowManager` 在语音输入期间订阅 `BaiduSpeechManager.noiseLevel`，输入结束时必须取消订阅，避免常驻：

```kotlin
private var noiseLevelJob: Job? = null

private fun onVoiceInputStarted() {
    noiseLevelJob?.cancel()
    noiseLevelJob = scope.launch {
        baiduSpeechManager.noiseLevel.collect { level ->
            handler.post { updateNoiseIndicator(level) }
        }
    }
}

private fun onVoiceInputEnded() {
    noiseLevelJob?.cancel()
    noiseLevelJob = null
    handler.post { circleView?.clearNoiseIndicator() }
}

private var lastHighStartMs = 0L
private fun updateNoiseIndicator(level: NoiseLevel) {
    when (level) {
        NoiseLevel.LOW -> {
            circleView?.setNoiseIndicator(Color.GREEN, "环境安静")
            lastHighStartMs = 0L
        }
        NoiseLevel.MEDIUM -> {
            circleView?.setNoiseIndicator(Color.YELLOW, "环境一般")
            lastHighStartMs = 0L
        }
        NoiseLevel.HIGH -> {
            circleView?.setNoiseIndicator(Color.RED, "环境嘈杂，建议换个地方")
            // 持续 3s HIGH 才提示，避免频繁打扰
            val now = System.currentTimeMillis()
            if (lastHighStartMs == 0L) lastHighStartMs = now
            else if (now - lastHighStartMs > 3000) {
                showSubtleToast("环境嘈杂可能影响识别")
                lastHighStartMs = now
            }
        }
    }
}
```

### 7.2 视觉位置

`CircleFloatingView` 录音状态下，录音图标右下角显示 8dp 圆点（绿/黄/红），不打断当前录音流程。

### 7.3 预学习期 UI

预学习 500ms 期间，悬浮窗显示"准备中..."而不是"正在听..."，避免用户误以为已经开始说话。

## 8. 可选增强

### 8.1 低功耗间歇采样（P2，唤醒场景）

**目标**：长时间无声环境下，把循环延迟从 10ms 提到 50ms，降低 CPU 占用。

```kotlin
class LowPowerSampler(
    private val normalDelayMs: Long = 10L,
    private val lowPowerDelayMs: Long = 50L,
    private val lowPowerSilenceMs: Long = 5000L
) {
    private var isLowPower = false
    private var lastActivityTime = System.currentTimeMillis()

    fun nextDelay(hasSpeech: Boolean): Long {
        val now = System.currentTimeMillis()
        if (hasSpeech) {
            lastActivityTime = now
            isLowPower = false
        } else if (now - lastActivityTime > lowPowerSilenceMs) {
            isLowPower = true
        }
        return if (isLowPower) lowPowerDelayMs else normalDelayMs
    }
}
```

**注意**：低功耗模式下 50ms 是经验值。再大会导致唤醒响应明显延迟（用户喊唤醒词后要等 ≥ 100ms 才被处理）。**不要超过 100ms**。

### 8.2 Silero VAD（P3，备选，不推荐主路径）

仅当 AdaptiveVad + SpectralNoiseDetector 在极端噪声下仍不达标时考虑。

**拒绝理由**：~2MB 包体积 + TFLite 运行时 + 异步加载复杂度。Vosk grammar 已经是强约束的人声过滤器，引入 Silero 边际收益低。

如确需引入，参考方案：assets 内置 `silero_vad.tflite`，首次复制到 `filesDir`，TFLite Interpreter 加载，输入 16kHz / 512 采样浮点 PCM，输出 [0,1] 语音概率。

### 8.3 场景切换自动重置（P2）

用户从室内走到街上后，原噪声基底失效。可加入突变检测：

```kotlin
// EnvironmentAnalyzer 内部
private var consecutiveHighEnergyFrames = 0
private fun checkEnvironmentShift(ste: Double) {
    // 单帧能量 > 基底 × 10 持续 60 帧（~2s）→ 可能切换了环境
    if (ste > adaptiveVad.getNoiseFloorSte() * 10) {
        consecutiveHighEnergyFrames++
        if (consecutiveHighEnergyFrames > 60) {
            Log.w(TAG, "检测到环境突变，重新预学习")
            startPreLearning()
            consecutiveHighEnergyFrames = 0
        }
    } else {
        consecutiveHighEnergyFrames = 0
    }
}
```

也可显式接入：`ChatViewModel` session 开始时调用 `environmentAnalyzer.reset()`。

## 11. 集成注意事项

1. **环境切换重置**：用户从室内走到街上后，可通过突变检测自动 `reset()`，或在 `ChatViewModel` session 开始时强制重置。
2. **与 `SettingsPrefs` 的关系**：暂不加"噪声灵敏度"手动选项（违背自动适应原则）；走全自动适应。
3. **UI 指示器生命周期**：只在录音期间显示，`onVoiceInputEnded()` 必须 `cancel()` 协程并清除视图，避免常驻。
4. **预学习期间的用户反馈**：500ms 内 UI 显示"准备中..."而非"正在听..."，避免用户误以为已开始说话。
5. **Vosk grammar 不能丢**：所有改造不影响 `WakeWordListeningManager` 对 Vosk grammar 的使用，grammar 是人声过滤的根本。
6. **线程模型**：`EnvironmentAnalyzer.processFrame` 在 `AudioRecord` 读取线程运行（与现有循环同线程，无需加锁）；`noiseLevel` StateFlow 跨线程订阅安全。
7. **预学习被打断**：如预学习期间 `audioRecord` 被外部 stop，应直接退出循环，下次 `start()` 重新预学习。
8. **预学习没采到帧**：极端情况下 2s 内一帧都没读到（设备故障），保持 `noiseFloor = 100.0` 作为 fallback，不要崩溃。

## 12. 备选方案与拒绝理由

| 备选方案 | 拒绝理由 |
|----------|----------|
| Silero VAD（深度学习） | 包体积 +2MB，Vosk grammar 已足够；保留为 P3 备选 |
| WebRTC VAD | JNI 绑定复杂，精度与纯算法相当 |
| Vosk 前做 Layer 1 过滤跳帧 | **错误**，Vosk 需要全帧输入维持状态机 |
| 设置页加手动噪声灵敏度 | 违背"自动适应"；用户难以估计环境噪声级别 |
| FFT 真实频域分析 | 移动端每帧 FFT 开销大；ZCR + 低频能量比是足够近似 |
| 只做能量检测不做频域 | 无法区分机械噪声/风声，无法给用户正确的 UI 提示 |
